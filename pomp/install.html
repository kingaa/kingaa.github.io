Installation

Development version

The development version is usually several weeks ahead of the version on CRAN. To use it, you will need to install it from source.

The easiest way to install the development version uses the devtools package. In an R session, do 

require(devtools)
install_github("kingaa/pomp")

Alternatively, you can download the source as a tarball or as a zip file. Unpack the tarball or zipfile (thereby creating a directory named, say, kingaa-pomp-xxxxxxx) and build the package source by executing 

R CMD build kingaa-pomp-xxxxxxx
Then install the package from the R command line: 
install.packages("pomp_X.X-X.tar.gz")

Stable version

Source and binaries for the CRAN version are available here.

Important note for Windows users

To compile the package from source, and to make use of its facilities for accelerated computation using compiled C code, you will need the ability to compile C code and dynamically link it into an R session. For this reason, you must install the Rtools suite, which can be downloaded from http://cran.r-project.org/bin/windows/Rtools. Rtools is needed both to compile and install the development version from source and to obtain full value from any version of the package.

Important note for Mac OS X users

To compile the package from source, and to be able to make use of its facilities for accelerated computation using compiled C code, you will need the ability to compile C code and dynamically link it into an R session. These facilities are provided in the Xcode app, which is free and can be installed via the App Store or downloaded from https://developer.apple.com/xcode/downloads/.

Some users report that gfortran is not installed by default in all versions of Xcode. For instructions on installing a suitable version of gfortran (and other libraries you may need), see http://cran.r-project.org/bin/macosx/tools for the R project's recommendations on development tools and libraries for Mac OS X.
